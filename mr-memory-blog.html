End the "Context Bloat" Era

AI development generates massive logs. Overloaded context windows slow down response times, spike costs, and degrade precision. mr-memory transforms these logs into a structured Knowledge Graph.

🔥 Hot Memory

Immediate session context. High visibility for active tasks. Resets every rotation to keep prompts lean.

⛅ Warm Memory

The distilled "Source of Truth". Contains persistent decisions, progress, and technical gotchas across sessions.

❄️ Cold Memory

Indexed historical archives. Stored off-context but instantly retrievable via hybrid BM25 search.

Key Capabilities

  • Bilingual Compaction: Automatically extracts tasks and decisions in IT/EN (e.g., Decisione:, ADR:, DONE:).
  • Token Auditing: Real-time estimation and health checks (Lean, Growing, Bloated) to monitor your budget.
  • Rotation Safety: Atomic resets and mandatory backups ensure you never lose a single byte of history.
  • Semantic Re-ranking: A two-phase protocol that combines lexical speed with LLM-level semantic precision.

The Pro Workflow

# 1. Analyze your current context weight
mr-memory audit --verbose

# 2. Distill session logs into the Warm Tier
mr-memory compact --backup

# 3. Archive history and restore performance
mr-memory rotate --keep-last 2

# 4. Find the needle in the Cold Memory haystack
mr-memory retrieve "SwiftData architecture"
